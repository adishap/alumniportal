    Let's get started...

    Step 1Plan the website

    A common problem with many Web projects: failure to plan.Planning is essential.
    Decide the aim, purpose and goal of the website.Collect all the requirements(reason to make that website) and from the requirments choose the idea which suits best.
    Sketch your Idea
    Get your brain around what kinds of tools and processes will accomplish your goals.Create a roadmap document that provides a broad overview of the process of developing a website, from the initial needs assessment through the launch, maintenance and follow-up.This is where we put all the ideas, literally everything into a giant to-do list.

    Step 2 Decide development Approach

    When done with the planning,we are going to analyze the system we are going to build.It requires these things to be completed:
    1. Decide techstack we are going to use.
    2. Developing a Workflow.
    Deciding the techstack
    Decide the technology which you are going to work upon.I used :
    • HTML
    • CSS
    • JavaScript
    • Bootstrap
    • PHP
    • MySQL
    For front-end, HTML, CSS, JavaScript and Bootstrap(it is the most popular HTML, CSS, and JavaScript framework) is used. There are number of frameworks available but I like bootstrap as it is completely free to download and use!
    For back-end, I made the choice to go with a combination of MySQL and PHP to handle all our behind-the-scenes data handling and storage.MySQL is a popular choice of database for use in web applications.

    Developing a Workflow
    As my websites have users, that means it needs a sign up form and a log in area for returning users.Because users should be just as concerned about security as I am, users need to be able to change their passwords, change their login, and delete their accounts. As I need to manage more than 10 webpages, so there is need to think about some workflow.
